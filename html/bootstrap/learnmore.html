What does Enhanced IP work with?

ssh, samba, http, and more.




How many addresses does Enhanced IP support?

Approximately, 2^56. Enough for all humans, the Internet of Things, and maybe even a remote planet or two.

What does an IPv4 Address look like?

192.0.2.2




What does an Enhanced IP Address look like?

192.0.2.2.10.0.0.1




192.0.2.2.10.0.0.1 looks like two IP addresses concatenated together. Is it that simple?

That's exactly what it is! The first IP is used to route the packet over the Internet while the second routes the packet over a second IP network. E.g. often your local LAN.




Is there space for this extra addressing in the IPv4 header?

Yes, if you use IP options. This is what Enhanced IP uses to get the extra address space. 12 extra bytes per packet to be exact.




Are IP options evil?

Some are rumored to be and some in fact are evil. We use IP option #26. It is not evil.




How does DNS work with Enhanced IP?

You can use your current DNS server and specify a AAAA record 192.0.2.2.10.0.0.1 like so:
      hostname IN AAAA 2001:0101:C000:0202:0a00:0001::0;




Doesn't the core router infrasture block some IP options packets.

This is true but we believe it can be undone with your help. See router report for more details. If you're a Cisco shop 'ip options ignore' makes us happy.




What about customer premises equipment?

If it's a NAT box, it will have to be patched. Our kernel patch is appoximately 400 lines of code. It is simple.




What exactly is Enhanced IP?

It's a patch to NAT that adds back in end-to-end addressing, it does this in such a way that no state has to be maintained on a per packet basis in the NAT. This makes it fast. It adds 12 extra bytes of data to each Enhanced IP packet. It uses IPv6 experimental DNS records for name lookup. It's a clever mashup of technologies that is still new and researchish. If you're trying to configure IPv6 and getting lost, it offers hope. If you're a hacker looking for something to work on, it offers a fun playground of challenging coding to do.




What about IPv6?

IPv6 requires repeering of BGP on a large scale. ~8500 AS's peered on IPv6, ~47470 AS's peered on IPv4 as of June 30, 2014




What about IPv6?

Customer premises equipment upgrades are necessary also. Their patch is more complex.




What about IPv6?

We can't help but think that the availability of good support for certain IP options across the Internet might make for a good IPv6 tunnelling protocol, one that doesn't require tunnel gateways.
